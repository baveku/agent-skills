# Planning

Follow `planning-and-task-breakdown`.

Read `SPEC.md` or the equivalent spec first. Identify dependencies, slice work vertically, write acceptance criteria and verification steps for each task, and save the output to `tasks/plan.md` plus `tasks/todo.md`.

Use `/planning` instead of `/plan` in Antigravity because `/plan` may be reserved by the host.
