# Spec

## Overview

Antigravity slash alias for the spec workflow.

## When to Use

- The user invokes `/spec`.
- A new project, feature, or significant change needs requirements.
- Requirements are unclear or need acceptance criteria before implementation.

## Process

1. Apply `rules/skill-routing.md`.
2. Follow `spec-driven-development`.
3. Stop for user confirmation before planning or implementation.
